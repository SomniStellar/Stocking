# 지수비교 결정사항

## 현재 구현 기준
- 기본 비교 세트는 `S&P 500`, `Nasdaq 100`, `Dow Jones`, `KOSPI`, `KOSDAQ`를 사용한다.
- 사용자 추가 지표는 미국 시장 티커만 허용한다.
- 사용자 추가 개수는 최대 3개로 제한한다.
- 동일 지수 또는 동일 티커의 중복 설정은 허용하지 않는다.
- Dashboard는 비교 카드와 기간 토글까지 구현했다.
- Settings에서 benchmark 활성화, custom 추가, 수정, 삭제를 지원한다.
- 기간은 `YTD`, `1Y`, `3Y`, `5Y`를 사용한다.

## 현재 데이터 기준
- 현재 구현은 `Monitor`, `Benchmarks`를 사용한다.
- Dashboard 비교 카드는 `Monitor` 스냅샷 값을 사용한다.
- `Monitor`는 그래프 단계 전까지 유지한다.

## 다음 구조 결정
- 그래프 단계에서는 `Quotes`, `SeriesCalendar`, `Series`를 추가한다.
- `YTD`는 당해연도 `DAILY` 시계열을 사용한다.
- `1Y`, `3Y`, `5Y`는 최근 5년 `WEEKLY` 시계열을 사용한다.
- 화면은 저장된 범위를 절단해서 사용한다.

## 벤치마크 조회 기준
- 직접지수를 primary 기준으로 사용한다.
- fallback은 해당 지수 인덱스 펀드 또는 ETF 후보로 둔다.
- 적절한 fallback 티커를 찾지 못한 경우 빈 값으로 둘 수 있다.
- 시트에는 fallback 후보 메타데이터만 저장한다.
- 실제 조회는 `resolved_ticker` 하나만 사용한다.
- primary 조회는 최대 2회 재시도한다.
- 재시도 후에도 실패하면 fallback이 있는 경우에만 fallback으로 전환한다.
- fallback 사용 시 UI에 대체 지표 사용 사실을 표시한다.
- fallback도 실패하면 메시지를 표시하고 해당 지표는 비교에서 제외한다.

## 계산 기준
- 정규화는 데이터처리 계층에서 수행한다.
- 정규화는 선택한 기간의 시작점 값을 기준으로 계산한다.
- 계산식은 `(point_value / base_value) * 100`이다.
- 시작점 이전 유효값이 없으면 해당 시리즈는 해당 기간 비교에서 제외한다.

## 날짜축 기준
- `SeriesCalendar`를 공통 기준 날짜축으로 사용한다.
- `CalendarDaily`는 당해연도 거래일 축이다.
- `CalendarWeekly`는 최근 5년 주간 축이며 기준일은 금요일이다.
- 축 날짜에 값이 없으면 가장 가까운 이전 유효값을 사용한다.
- 허용 간격은 `DAILY` 7일, `WEEKLY` 14일이다.

## 실패 처리 기준
- 지표 로드 실패 시 카드/그래프 근처에 짧은 캡션을 표시한다.
- fallback 사용 시 대체 지표 사용 상태를 표시한다.
- 실패한 시리즈는 카드와 그래프 계산에서 제외한다.
