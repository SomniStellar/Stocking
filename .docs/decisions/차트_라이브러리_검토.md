# 차트 라이브러리 검토

## 목적
- 차트 1차 범위에 맞는 라이브러리 선택 근거를 정리한다.
- 현재 결론은 `지수비교_결정사항.md`의 기능 범위 기준과 같이 본다.

## 검토 기준
현재 요구 기능은 아래 4가지다.
- 선택한 시점의 주가액 표출
- 시계열 범위 확대 또는 축소
- 극점 및 고저점 값 상시 표출
- 비교 그래프로서의 시각 완성도

## Recharts 적합도
### 가능한 항목
- 선택 시점 값 표출: 가능
  - `Tooltip`과 커스텀 콘텐츠로 현재 커서 또는 선택 지점의 값을 표시할 수 있다.
- 범위 확대 또는 축소: 가능
  - `Brush`로 X축 범위 절단과 이동이 가능하다.
- 극점 또는 고저점 표시: 가능
  - 데이터처리 계층에서 극점을 계산한 뒤 `ReferenceDot`, `ReferenceLine`, `LabelList`, `Customized` 조합으로 상시 표출할 수 있다.
- 반응형 배치: 가능
  - `ResponsiveContainer` 기반으로 화면 폭 대응이 가능하다.

### 주의할 항목
- 확대 또는 축소는 기본적으로 `Brush` 중심이다.
- 마우스휠 확대, 드래그 줌, 자유로운 팬은 기본 제공 강점이 아니다.
- 고저점 라벨은 별도 계산과 커스텀 렌더링이 필요하다.
- 기본 스타일만 쓰면 시각적으로 평범해 보일 가능성이 높다.

### 시각 품질 평가
- 기본 컴포넌트만 쓰면 충분히 기능적이지만, 시각 완성도는 보통 수준이다.
- 색상, 라인 굵기, 활성 포인트, 기준선, 툴팁, 라벨 밀도 조절을 직접 설계하면 서비스용으로 충분히 깔끔하게 만들 수 있다.
- 현재 프로젝트 데이터량은 작아서 성능 이슈보다는 상호작용 밀도와 시각 디테일이 관건이다.

## ECharts 비교
### 강점
- 확대 또는 축소와 상호작용은 더 강하다.
- 상시 라벨, 강조점, 보조선, 시각 효과 구현 폭이 넓다.
- PC와 모바일 최적화, 복합 상호작용에는 여유가 있다.

### 단점
- 도입 비용이 더 크다.
- 현재 프로젝트 1차 범위에는 과한 선택일 수 있다.
- React 프로젝트에 바로 붙이는 비용이 Recharts보다 높다.

## 현재 결론
- 현재 구현 예정인 1차 그래프 범위는 `기본 선형 그래프 + 선택 시점 값 표시`로 둔다.
- 이 범위에서는 Recharts로 충분하다.
- `Brush` 기반 범위 조절과 극점 표시는 후속 단계에서 추가 검토한다.

## 재검토 조건
- 마우스휠 줌, 자유 팬, 고밀도 상호작용까지 요구되면 ECharts를 다시 검토한다.
