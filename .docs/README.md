# .docs

## 구조
- `current/`: 현재 기준 본편 문서
- `decisions/`: 결정 완료 사항과 미결정 사항 문서
- `guides/`: 설정 및 테스트 가이드

## 시작 문서
- 사용자 검토 시작점은 `current/문서_마스터맵.md`다.
- 개별 문서를 바로 열기 전에 먼저 마스터맵에서 현재 기준 경로를 확인한다.

## 분리 원칙
- `.docs`는 사용자 검토용 문서만 둔다.
- `.codex`는 Codex 참조용 문서만 둔다.
- 같은 주제를 `.docs`와 `.codex`에 중복 작성하지 않는다.
- 예외는 `.codex`에만 필요한 운영 메모뿐이다.

## `.docs`에 들어갈 내용
- 기능 범위
- 화면 명세
- 시트 구조
- 결정 완료 사항
- 추가 검토가 필요한 사용자 판단 항목
- 설정 및 테스트 가이드

## `.docs`에 들어가면 안 되는 내용
- 에이전트 작업 절차
- 작업 중간 메모
- 커밋 처리 순서 같은 내부 운영 메모
- 어떤 문서를 먼저 갱신할지에 대한 에이전트용 실행 지시
- 임시 판단, 초안 메모, 정리 전 잔여 텍스트

## 문서 처리 기준
- 현재 기준 내용은 `current/`에만 둔다.
- 결정 완료 사항과 추가 검토 필요 사항은 같은 문서에 섞지 않는다.
- 가이드는 `guides/`에만 둔다.
- 파일 경로는 고정 경로로 유지한다.
- 새 버전 파일을 계속 늘리지 말고 현재 기준 파일을 갱신한다.

## 금지 기준
- `.docs`에 Codex 전용 작업 메모를 쓰지 않는다.
- `.codex`에 사용자 검토용 설계 본편을 쓰지 않는다.
- 결정 완료 사항을 `미결정사항` 문서에 남겨두지 않는다.
- 본편 변경 후 마스터맵 갱신을 누락하지 않는다.
