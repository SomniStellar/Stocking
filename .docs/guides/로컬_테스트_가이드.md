# 로컬 테스트 가이드 v4

## 목적
- 현재 프로젝트를 로컬에서 실행해서 화면과 기본 기능을 확인한다.
- Google 로그인 테스트 시 승인된 JavaScript 원본 주소와 동일한 고정 주소를 사용한다.
- Holdings 수동 정렬과 문서 현행화 이후 핵심 회귀 포인트를 함께 점검한다.

## 실행 위치
- 프로젝트 루트: `E:\WorkSpace\Stocking`

## 고정 로컬 주소
- 개발 서버 주소: `http://127.0.0.1:4173/Stocking/`
- Preview 주소: `http://127.0.0.1:4173/Stocking/`
- Google Cloud OAuth 승인된 JavaScript 원본 주소에는 아래 origin을 등록한다.
  - `http://127.0.0.1:4173`

## 기본 실행 순서
1. 터미널을 프로젝트 루트에서 연다.
2. 의존성을 설치한다.
3. 개발 서버를 실행한다.
4. 브라우저에서 `http://127.0.0.1:4173/Stocking/` 로 접속한다.

## 명령어
일반 환경:
```bash
npm install
npm run dev
```

Windows PowerShell에서 `npm`이 바로 실행되지 않으면:
```powershell
npm.cmd install
npm.cmd run dev
```

## 주소 확인 방법
- 현재 프로젝트는 `vite.config.ts`에서 host와 port를 고정했다.
- 따라서 정상 실행되면 접속 주소는 항상 `http://127.0.0.1:4173/Stocking/` 이다.
- 터미널 출력의 `Local:` 항목으로도 다시 확인할 수 있다.
- 만약 `4173` 포트를 다른 프로그램이 사용 중이면, Vite는 다른 포트로 넘어가지 않고 실행을 중단한다.
- 이 경우 포트를 점유한 프로그램을 종료한 뒤 다시 실행한다.

## Holdings 확인 포인트
- Add slot 카드가 데스크톱과 좁은 레이아웃에서 시각적으로 중앙 정렬되어 보인다.
- Holdings 카드에서 티커와 수량 칸이 과도하게 넓지 않고, 가격/손익/평가금액 칸이 상대적으로 넓다.
- Holdings 카드 드래그 앤 드롭으로 순서를 바꿀 수 있다.
- 태그 필터가 적용된 상태에서도 보이는 카드만 재정렬되고, 필터를 해제하면 숨겨진 카드의 상대 순서는 유지된다.
- 페이지 새로고침 후에도 카드 순서가 유지된다.
- Holdings 수정/삭제 후에도 기존 카드 순서가 크게 흐트러지지 않는다.

## 기본 확인 포인트
- 로그인 페이지가 열린다.
- Google 로그인 버튼이 보인다.
- 로그인 후 설정 화면에서 템플릿 시트를 만들 수 있다.
- Holdings 화면에서 BUY / SELL 입력 폼이 보인다.

## 참고
- `vite.config.ts`의 `base` 설정은 `/Stocking/` 이다.
- 실제 Google 로그인 테스트를 하려면 `.env.local`에 `VITE_GOOGLE_CLIENT_ID`가 설정되어 있어야 한다.
