# 로컬 테스트 가이드

## 목적
- 현재 프로젝트를 로컬에서 실행하고 화면, 기본 동작, preview 동작을 검증한다.
- UI 검증용 preview와 실제 Google Sheets 연동 테스트를 구분해서 사용한다.

## 작업 위치
- 프로젝트 루트: `E:\WorkSpace\Stocking`

## 기본 실행 주소
- 개발 서버 기본 주소: `http://127.0.0.1:4173/`
- 현재 앱은 `HashRouter`를 사용하므로 실제 화면 경로는 `/#/...` 형식이다.

## Preview 테스트 방식
### 목적
- 로그인 없이 샘플 데이터로 화면 레이아웃과 기본 상호작용을 확인한다.
- Dashboard, Holdings, Settings, Watchlists 기본 구조를 빠르게 검증한다.

### 동작 조건
- 호스트는 `127.0.0.1` 또는 `localhost`여야 한다.
- 주소에 `?preview=...` 쿼리가 있어야 한다.
- 허용 preview 값은 `dashboard`, `holdings`, `settings`, `watchlists`다.

### 주소 예시
- Dashboard preview: `http://127.0.0.1:4173/?preview=dashboard#/dashboard`
- Holdings preview: `http://127.0.0.1:4173/?preview=holdings#/holdings`
- Settings preview: `http://127.0.0.1:4173/?preview=settings#/settings`
- Watchlists preview: `http://127.0.0.1:4173/?preview=watchlists#/watchlists`

### Preview에서 확인되는 동작
- 로그인 없이 해당 화면으로 바로 진입한다.
- 실제 Google Sheets 대신 샘플 snapshot 데이터를 사용한다.
- Dashboard에서는 기본 benchmark 카드와 `Add Benchmark` 입력 UI를 확인할 수 있다.
- Holdings에서는 `Portfolio` 카드와 `Add Slot` 카드, holding 카드 배치를 확인할 수 있다.
- Settings에서는 계정/연결 설정 레이아웃만 확인한다.
- Watchlists에서는 빈 상태 또는 현재 샘플 기준의 기본 레이아웃만 확인한다.

### Preview에서 확인할 것
- 레이아웃
- 반응형 배치
- 카드 너비와 정렬
- 입력 UI 열림 상태
- 기본 문구와 아이콘 상태
- Playwright smoke 테스트용 캡처 생성

### Preview에서 확인되지 않는 것
- 실제 Google 로그인 성공 여부
- 실제 스프레드시트 생성/연결/수정
- 실제 benchmark 저장과 Sheets 반영
- 실제 fallback 전환
- 실제 데이터 지연 로드 실패

## 실연결 테스트 방식
### 목적
- 실제 Google 계정, 실제 Sheets, 실제 저장 동작을 확인한다.

### 선행 조건
- `.env.local`에 `VITE_GOOGLE_CLIENT_ID`가 설정되어 있어야 한다.
- Google Cloud OAuth 설정에 현재 로컬 origin이 등록되어 있어야 한다.
  - `http://127.0.0.1:4173`
  - 필요 시 `http://localhost:4173`

### 주소
- 로그인: `http://127.0.0.1:4173/#/login`
- Dashboard: `http://127.0.0.1:4173/#/dashboard`
- Holdings: `http://127.0.0.1:4173/#/holdings`
- Watchlists: `http://127.0.0.1:4173/#/watchlists`
- Settings: `http://127.0.0.1:4173/#/settings`

### 실연결에서 확인할 항목
- Google 로그인 성공 여부
- Settings에서 스프레드시트 생성/연결/리셋
- Holdings 추가/수정/삭제/정렬 저장
- Watchlists 추가/수정/삭제 저장
- Dashboard benchmark 추가/수정/삭제/활성화 토글
- `Monitor`, `Benchmarks` 시트 동기화
- benchmark 카드 표시/비표시 상태

## Dashboard 체크리스트
- 개발/테스트 문구는 필요한 경우에만 `[Dev/Test]`로 표시되는지 확인
- 상단 포트폴리오 요약은 충분한 폭에서 `1 x 4`, 중간 폭에서 `2 x 2`로만 전환되는지 확인
- 상단 요약 섹터와 benchmark 섹터가 가로 중앙 정렬인지 확인
- 상단 요약과 benchmark 카드가 공용 fixed-grid 규칙대로 동작하는지 확인
- benchmark 영역이 `4 -> 3 -> 2 -> 1`로 줄어들고, 줄어들기 전에 가로 스크롤이 생기지 않는지 확인
- 첫 카드가 `Portfolio` 카드인지 확인
- benchmark 카드의 delta 라벨이 `vs Port.`인지 확인
- 카드 전체 클릭으로 활성/비활성 토글이 동작하는지 확인
- `Add Benchmark`가 제목 옆에서 compact `Ticker + Save + Cancel` UI로 열리는지 확인

## Holdings 체크리스트
- 상단 섹터는 `Portfolio` 카드와 `Add Holding` 카드만 포함하는지 확인
- 하단 섹터는 holding 카드만 포함하는지 확인
- 두 섹터가 같은 고정폭 카드 시스템을 공유하는지 확인
- 카드가 `2 x 3` 구조로 보이는지 확인
- `Total Price` <-> `Avg Price` 토글이 클릭 가능하게 보이는지 확인
- `BUY` <-> `SELL` 토글이 저장 버튼과 충돌 없이 정렬되는지 확인
- 태그, 가격, 수익률, 기간 블록이 과도하게 겹치지 않는지 확인

## Watchlists 체크리스트
- 페이지 전용 CSS 없이 공용 카드/폼 규칙만으로 정상 배치되는지 확인
- 상단 입력 폼이 `2열 -> 1열` 반응형으로 전환되는지 확인
- 카드 헤더와 compact metric grid가 공용 패턴으로 정상 표시되는지 확인
- 이후 Holdings 유사 구조로 확장하더라도 현재 기본 배치가 깨지지 않는지 확인

## 주의사항
- Preview는 실연결 테스트가 아니다.
- 먼저 주소 형식이 `?preview=...#/route`인지 확인한다.
- 다음으로 호스트가 `127.0.0.1` 또는 `localhost`인지 확인한다.
- 실제 저장/연동 동작은 preview가 아니라 실연결 테스트에서 확인한다.
