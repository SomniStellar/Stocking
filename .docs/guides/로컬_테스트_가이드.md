# 로컬 테스트 가이드

## 목적
- 현재 활성 UI 범위인 `Login`, `Dashboard`, `Holdings`, `Settings`를 로컬에서 확인한다.
- preview 확인과 실제 Google Sheets 연동 확인을 구분한다.

## 작업 위치
- 프로젝트 루트: `E:\WorkSpace\Stocking`

## 기본 실행
### 의존성 설치
```powershell
npm.cmd install
```

### 개발 서버
```powershell
npm.cmd run dev
```

### 프리뷰 서버
```powershell
npm.cmd run preview
```

### 정적 검증
```powershell
npm.cmd run lint
npm.cmd run build
```

### E2E
```powershell
npm.cmd run test:e2e
```

## Preview 확인
### 목적
- 로그인 없이 샘플 데이터로 화면 구조와 반응형 레이아웃을 빠르게 확인한다.

### 주소 예시
- Dashboard: `http://127.0.0.1:4173/?preview=dashboard#/dashboard`
- Holdings: `http://127.0.0.1:4173/?preview=holdings#/holdings`
- Settings: `http://127.0.0.1:4173/?preview=settings#/settings`

### Preview에서 확인할 것
- Dashboard 카드 정렬과 차트 영역 기본 구조
- Holdings 카드 정렬, `Add Holding` 입력 카드, 태그 필터
- Settings 계정/시트 액션 레이아웃

### Preview에서 확인되지 않는 것
- 실제 Google 로그인 성공 여부
- 실제 시트 생성/연결/저장
- 실제 benchmark fallback 전환
- 실제 시계열 로드 실패나 권한 오류

## 실연결 확인
### 목적
- 실제 Google 계정과 실제 Google Sheets 기준으로 주요 동작을 확인한다.

### 선행 조건
- `.env.local`에 `VITE_GOOGLE_CLIENT_ID`가 설정돼 있어야 한다.
- Google Cloud OAuth Origin에 현재 로컬 주소가 등록돼 있어야 한다.
  - `http://127.0.0.1:4173`
  - 필요 시 `http://localhost:4173`

### 주요 경로
- Login: `http://127.0.0.1:4173/#/login`
- Dashboard: `http://127.0.0.1:4173/#/dashboard`
- Holdings: `http://127.0.0.1:4173/#/holdings`
- Settings: `http://127.0.0.1:4173/#/settings`

### 실연결에서 확인할 것
- Google 로그인 성공 여부
- Settings에서 새 시트 생성 또는 기존 시트 연결
- `Sync latest data`, `Rebuild sheet data`, `Open sheet`, `Disconnect sheet` 동작
- Holdings 추가/수정/삭제/정렬
- Dashboard benchmark 추가/삭제/활성화 전환
- 초기 로딩 후 benchmark 카드와 차트 반영 순서

## 화면 체크리스트
### Dashboard
- 상단 요약 카드가 `1 x 4` 또는 `2 x 2`로만 정렬되는지 확인
- benchmark 카드가 `4 -> 3 -> 2 -> 1`로 줄어드는지 확인
- 첫 비교 카드가 `Portfolio`인지 확인
- `vs Port.` 값이 `benchmark return - portfolio return`인지 확인
- benchmark 카드를 클릭했을 때 활성/비활성 상태가 자연스러운지 확인
- `Add Benchmark`가 섹션 제목 줄에서 compact하게 열리는지 확인

### Holdings
- 상단은 `Portfolio`와 `Add Holding` 두 카드만 보이는지 확인
- 하단은 holding 카드만 보이는지 확인
- `Total Price` <-> `Avg Price` 토글이 한 줄 버튼형으로 보이는지 확인
- 수량은 최대 4자리, 가격은 최대 2자리로 보이는지 확인
- `Total Price` <-> `Avg Price` 전환 시 총액이 표시 반올림만으로 깨지지 않는지 확인
- 드래그 정렬이 원하는 순서로 저장되는지 확인

### Settings
- 계정 섹션이 불필요한 상태 배지 없이 단순한지 확인
- Spreadsheet 섹션의 액션 버튼이 데스크톱에서 한 줄로 정렬되는지 확인
- `Disconnect sheet`가 현재 연결 해제 의미로 읽히는지 확인

## 참고
- preview는 실연결 테스트가 아니다.
- 실저장과 권한 문제는 반드시 실연결 상태에서 확인한다.
