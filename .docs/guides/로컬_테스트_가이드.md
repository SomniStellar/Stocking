# 로컬 테스트 가이드

## 목적
- 현재 프로젝트를 로컬에서 실행하고 화면, 저장 동작, preview 동작을 검증한다.
- UI 점검용 preview와 실제 Google Sheets 연동 테스트를 구분해서 사용한다.

## 작업 위치
- 프로젝트 루트: `E:\WorkSpace\Stocking`

## 기본 실행 주소
- 개발 서버 기본 주소: `http://127.0.0.1:4173/`
- Preview 서버 기본 주소: `http://127.0.0.1:4173/`
- 현재 앱은 `HashRouter`를 사용하므로 실제 화면 경로는 `/#/...` 형식이다.

## 기본 실행 명령

## Preview 테스트 방식

### 목적
- 로그인 없이 샘플 데이터로 화면 레이아웃과 기본 상호작용을 확인한다.
- Dashboard, Holdings, Settings의 기본 구조와 UI 회귀를 빠르게 점검한다.

### 동작 조건
- 호스트가 `127.0.0.1` 또는 `localhost`여야 한다.
- 주소의 해시 앞에 `?preview=...` 쿼리가 있어야 한다.
- 허용 preview 값은 현재 `dashboard`, `holdings`, `settings`, `watchlists` 이다.

### 주소 예시
- Dashboard preview: `http://127.0.0.1:4173/?preview=dashboard#/dashboard`
- Holdings preview: `http://127.0.0.1:4173/?preview=holdings#/holdings`
- Settings preview: `http://127.0.0.1:4173/?preview=settings#/settings`
- Watchlists preview: `http://127.0.0.1:4173/?preview=watchlists#/watchlists`

### Preview에서 기대하는 동작
- 로그인 화면으로 이동하지 않고 바로 해당 화면이 열린다.
- 실제 Google Sheets 대신 샘플 snapshot 데이터가 사용된다.
- Dashboard에서는 기본 benchmark 카드와 `Add Benchmark` 카드가 보인다.
- Holdings에서는 `Portfolio` 카드와 `Add Slot` 카드가 보인다.
- Settings에서는 계정/연결 설정 화면만 확인한다.

### Preview에서 확인할 수 있는 것
- 레이아웃
- 반응형 배치
- 카드 클릭/입력 UI 열림
- 기본 문구/아이콘/상태 표시
- Playwright smoke 테스트와 캡처 생성

### Preview에서 확인할 수 없는 것
- 실제 Google 로그인 성공 여부
- 실제 스프레드시트 생성/연결/저장
- 실제 benchmark 저장 후 Sheets 반영
- 실제 fallback 전환
- 실데이터 기준 로드 실패

## 실연결 테스트 방식

### 목적
- 실제 Google 계정, 실제 Sheets, 실제 저장/동기화 동작을 확인한다.

### 선행 조건
- `.env.local`에 `VITE_GOOGLE_CLIENT_ID`가 설정되어 있어야 한다.
- Google Cloud OAuth 설정에 현재 로컬 origin이 등록되어 있어야 한다.
  - `http://127.0.0.1:4173`
  - 필요 시 `http://localhost:4173`

### 주소
- 로그인: `http://127.0.0.1:4173/#/login`
- Dashboard: `http://127.0.0.1:4173/#/dashboard`
- Holdings: `http://127.0.0.1:4173/#/holdings`
- Settings: `http://127.0.0.1:4173/#/settings`

### 실연결에서 확인할 항목
- Google 로그인 성공 여부
- Settings에서 스프레드시트 생성/연결/리셋
- Holdings 추가/수정/삭제/정렬 저장
- Dashboard benchmark 추가/수정/삭제/활성화 저장
- `Monitor`, `Benchmarks` 시트 동기화
- 실데이터 기준 benchmark 카드 표시/제외

## Dashboard 확인 체크리스트
- 불필요한 사용자 문구가 없는지 확인
- 개발/테스트 문구는 `[Dev/Test]` 표시가 있는지 확인
- `Portfolio` 기준 카드가 첫 카드로 보이는지 확인
- benchmark 카드가 `항목명 | 변동률`, `vs Portfolio | 변동률` 구조로 보이는지 확인
- benchmark 카드 전체 클릭으로 활성화/비활성화가 되는지 확인
- 비활성 카드가 희미해지되 클릭 가능성이 유지되는지 확인
- `Add Benchmark` 카드 전체 클릭 시 ticker 입력창이 열리는지 확인

## Holdings 확인 체크리스트
- `Portfolio` 카드와 일반 holdings 카드 높이가 과도하게 깨지지 않는지 확인
- 일반 holdings 카드가 2행 3열 구조로 보이는지 확인
- 첫 열을 제외한 나머지 열 폭이 과도하게 다르지 않은지 확인
- 라벨이 상단 정렬로 보이는지 확인
- `Price / Avg`, `Value / Invested`가 균형 있게 배치되는지 확인
- `P/L` 금액과 변화율이 같은 강도로 보이는지 확인
- `Average Price`, `Total Price` 전환이 클릭 가능하게 보이는지 확인
- 전환 시 입력값이 평균가/총액 기준으로 함께 바뀌는지 확인
- 태그가 대소문자와 무관하게 합쳐지는지 확인

## 주의사항
- Preview에서 로그인으로 넘어가면 정상 동작이 아니다.
- 먼저 주소 형식이 `?preview=...#/route` 인지 확인한다.
- 다음으로 호스트가 `127.0.0.1` 또는 `localhost` 인지 확인한다.
- 실제 저장 동작은 preview가 아니라 실연결 테스트에서 확인한다.
