# 시트 템플릿 설계서

## 목적
현재 구현된 Google Sheets 탭 구조와, 다음 단계 시계열 확장 구조를 구분해서 관리한다.

## 현재 기준 구조
현재 구현은 `Holdings + Watchlists + Monitor + Benchmarks` 구조를 기준으로 한다.

## 다음 확장 구조
그래프 단계에서는 `Quotes + SeriesCalendar + Series` 구조를 추가한다.

## 현재 탭 구성
- Holdings
- Watchlists
- Monitor
- Benchmarks

## 다음 탭 구성
- Quotes
- SeriesCalendar
- Series

## Holdings
목적:
- 보유 종목 원본 데이터 저장

주요 컬럼:
- ticker
- name
- quantity
- avg_price
- tags
- note
- display_order
- updated_at

주요 컬럼:

- name
- target_price
- tags
- note
- updated_at

## Monitor
목적:
- 현재 구현 기준 카드 계산용 가격 스냅샷 저장

주요 컬럼:
- ticker
- market
- name
- current_price
- close_yesterday
- ytd_price
- price_1y
- price_3y
- price_5y
- updated_at

규칙:
- 현재 Dashboard 비교 카드와 Holdings 계산은 `Monitor`를 기준으로 한다.
- `Monitor`는 현재 구현 유지 대상이며, 그래프 단계 전까지 삭제하지 않는다.

## Benchmarks
목적:
- 주요지표 비교 대상 정의

주요 컬럼:
- benchmark_key
- ticker_primary
- ticker_fallback
- resolved_ticker
- resolved_source
- status
- market
- name
- category
- is_default
- is_enabled
- display_order
- retry_count

규칙:
- 기본 고정 세트는 `S&P 500`, `Nasdaq 100`, `Dow Jones`, `KOSPI`, `KOSDAQ`를 사용한다.
- 사용자 추가 티커는 미국 시장만 허용한다.
- 사용자 추가 티커는 최대 3개로 제한한다.
- 동일 지수 또는 동일 티커의 중복 설정은 허용하지 않는다.
- `ticker_primary`는 직접지수 기준 티커다.
- `ticker_fallback`은 해당 지수 인덱스 펀드 또는 ETF 후보 티커다.
- fallback 후보를 정하지 못한 경우 `ticker_fallback`은 비워둘 수 있다.
- `resolved_ticker`는 실제 조회에 사용하는 현재 활성 티커다.
- `resolved_source`는 `primary`, `fallback` 중 하나다.
- `status`는 `ready`, `retrying`, `fallback`, `failed` 중 하나다.
- `retry_count`는 primary 재시도 횟수 추적에 사용한다.

주요 컬럼:

- market
- name
- current_price
- close_yesterday
- ytd_price
- price_1y
- price_3y
- price_5y
- updated_at

규칙:

- `Monitor`를 대체하거나 분리하는 다음 단계 구조다.

## SeriesCalendar
목적:
- 그래프와 포트폴리오 합성을 위한 기준 날짜축 관리

주요 컬럼:
- calendar_key
- calendar_type
- point_date
- week_anchor
- period_scope
- updated_at

규칙:
- `calendar_type`은 `DAILY`, `WEEKLY`
- `period_scope`는 `YTD`, `LONG`
- `CalendarDaily`는 당해연도 거래일만 생성한다.
- `CalendarWeekly`는 최근 5년 범위의 주간 기준일만 생성한다.
- 주간 기준일은 금요일로 고정한다.
- `SeriesCalendar`는 앱/동기화 루틴이 생성한다.

## Series
목적:
- 포트폴리오 및 지표 비교용 시계열 저장

주요 컬럼:
- series_key
- series_type
- ticker
- name
- sample_type
- point_date
- point_value
- updated_at

규칙:
- `series_type`은 `portfolio`, `benchmark`
- `sample_type`은 `DAILY`, `WEEKLY`
- `DAILY`는 당해연도 범위만 저장하고 `YTD`에만 사용한다.
- `WEEKLY`는 최근 5년 범위만 저장하고 `1Y`, `3Y`, `5Y`에 사용한다.
- `Series`는 `SeriesCalendar`의 날짜축에 맞춰 정렬한다.
- 기준축 날짜에 값이 없으면 가장 가까운 이전 유효값을 사용한다.
- 허용 간격은 `DAILY` 7일, `WEEKLY` 14일이다.
- 허용 간격을 초과하면 해당 시점은 결측으로 처리한다.

생성 방식:
- Benchmark 기준 시계열은 `resolved_ticker` 기준 계산식 저장 / 자동생성을 사용한다.
- fallback 시계열은 상시 적재하지 않는다.
- primary 실패 시 최대 2회 재시도 후 fallback이 있으면 `resolved_ticker`를 fallback으로 전환한다.
- Portfolio 기준 시계열은 데이터처리 계층에서 계산한다.

정규화 규칙:
- 정규화 값은 저장하지 않는다.
- 정규화는 데이터처리 계층에서 선택 기간 시작점 기준으로 계산한다.
- 계산식은 `(point_value / base_value) * 100`이다.

디버그 규칙:
- 디버그 export가 필요하면 별도 플래그로 제어한다.
- 디버그 export는 기본값 `OFF`이다.
- 디버그 export 결과는 실서비스의 기준 데이터로 사용하지 않는다.
- 디버그 export는 제거가 쉬운 구조로 유지한다.
