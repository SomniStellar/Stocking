# 대시보드 차트 개발 정리

## 목적
- 대시보드 benchmark 비교 차트의 현재 구현 상태와 후속 항목을 정리한다.

## 현재 구현 상태
- `Recharts` 기반 benchmark 비교 차트 구현 완료
- 기간 선택은 `YTD`, `3Y`, `5Y`
- 기간 버튼은 전체 차트 범위를 전환한다
- 차트는 benchmark 카드 아래에 배치된다
- 차트 폭은 benchmark 카드 grid와 같은 centered fixed-width 규칙을 따른다
- hover는 선택 범위 내 해당 시점 값을 보여준다
- `failed` benchmark는 카드에는 남고 차트 비교에서는 제외된다

## 현재 데이터 기준
- 대시보드 live 필수 탭은 `Holdings`, `Monitor`, `Benchmarks`, `SeriesCalendar`, `Series`
- 첫 로딩에서는 `Monitor`를 먼저 동기화하고 `Series`는 background로 보강한다
- 그래서 benchmark 카드는 차트보다 먼저 보일 수 있다

## 포트폴리오 시계열 전제
- 현재 프로젝트는 거래이력 모델이 아니다
- 포트폴리오 시계열은 현재 `Holdings` 기준 계산값이다
- 과거 특정 시점 값은 실제 체결 이력 반영 성과가 아니라 현재 보유 수량 기준 추정값이다

## benchmark 규칙
- 기본 benchmark는 `S&P 500`, `Nasdaq 100`, `Dow Jones`, `KOSPI`, `KOSDAQ`
- custom benchmark는 최대 3개
- custom benchmark 카드는 현재 `추가 / 삭제`만 지원
- benchmark 활성화/비활성화는 카드 클릭으로 처리한다
- fallback까지 실패하면 `status=failed`로 보고 차트 비교에서 제외한다

## 후속 구현 후보
- 상단 포트폴리오 전용 수익금 그래프
- 번들 크기 최적화
- `DashboardComparisonChart` lazy mount 또는 deferred mount는 현재 단계에서는 보류
- 차트 shell 높이 유지와 skeleton/empty frame 정리는 lazy/deferred mount를 다시 검토할 때 함께 본다

## 미결정 항목
- `KOSPI`, `KOSDAQ` fallback 후보 ticker
- fallback 사용 문구 기준
- 오류 caption 문구 기준
- 포트폴리오 계산 검증 기준
