# 대시보드 차트 개발 정리

## 목적
- 대시보드의 benchmark 비교 차트 구현 상태와 현재 기준을 한 문서에서 확인한다.
- 구현 완료 범위와 후속 대기 항목을 분리해서 관리한다.

## 현재 구현 상태
- 대시보드 비교 차트는 `Recharts` 기반으로 구현되어 있다.
- 기간 토글은 `YTD`, `3Y`, `5Y`를 사용한다.
- 기간 버튼은 여러 기간을 동시에 보여주는 요약이 아니라 차트 전체 범위를 전환하는 토글이다.
- 차트는 benchmark 카드 섹션 바로 아래에 배치한다.
- 차트 폭은 benchmark 카드 그리드와 같은 centered fixed-width 규칙을 따른다.
- hover는 선택한 기간 범위 안의 해당 시점 값을 보여준다.
- `failed` benchmark는 카드에는 남아도 차트 비교에서는 자동 제외된다.

## 현재 데이터 기준
- 현재 live 구조는 `Holdings`, `Watchlists`, `Monitor`, `Benchmarks`, `SeriesCalendar`, `Series`를 사용한다.
- 첫 로딩에서는 `Monitor`를 먼저 동기화하고 `Series`는 background로 보강한다.
- 그래서 초기 진입 직후에는 benchmark 카드가 먼저 보이고 차트는 잠깐 `pending` 상태일 수 있다.

## 포트폴리오 시계열 기준
- 현재 프로젝트는 거래이력 모델이 아니라 `현재 보유 상태` 모델이다.
- 현재 포트폴리오 시계열은 거래이력 기반 확정 성과가 아니라 현재 `Holdings` 기준 계산값이다.
- 따라서 과거 특정 시점의 포트폴리오 값은 실제 체결 이력 반영 결과가 아니라 현재 보유 수량 기준 추정값일 수 있다.

## benchmark 규칙
- 기본 benchmark 세트는 `S&P 500`, `Nasdaq 100`, `Dow Jones`, `KOSPI`, `KOSDAQ`다.
- custom benchmark는 최대 3개까지 허용한다.
- custom benchmark 카드는 현재 `추가/삭제`만 지원한다.
- benchmark 활성화/비활성화는 카드 클릭으로 처리한다.
- fallback까지 실패하면 `status=failed`로 보고 차트 비교에서 제외한다.

## 현재 남은 구현 후보
- `DashboardComparisonChart` lazy mount 또는 deferred mount
- 차트 shell 높이 유지와 skeleton/empty frame 정리
- 상단 포트폴리오 전용 수익금 그래프
- 번들 크기 최적화

## 미결정 항목
- `KOSPI`, `KOSDAQ` fallback 후보 ticker
- fallback 사용 문구 기준
- 오류 caption 문구 기준
- 포트폴리오 계산 검증 기준
