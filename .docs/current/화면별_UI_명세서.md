# 화면별 UI 명세서

## Dashboard
목적:
- 포트폴리오 요약과 주요지표 비교를 한 화면에서 제공한다.

현재 구성:
- 요약 카드
- 비교 카드
- 기간 토글: `YTD`, `1Y`, `3Y`, `5Y`
- 로드 실패 메시지
- fallback 사용 상태 문구

다음 구성:
- 시계열 선형 그래프

규칙:
- 비교 카드를 그래프보다 먼저 배치한다.
- 현재는 비교 카드만 구현 기준으로 본다.
- 로드 실패한 시리즈는 카드 계산에서 제외한다.
- 사용자 추가 지표 선택 UI는 Settings에서 관리한다.
- 그래프와 확대/축소 UI는 2차 범위로 둔다.

## Holdings
목적:
- 보유 종목과 포트폴리오 요약을 조회, 편집, 정렬한다.

구성:
- 포트폴리오 요약 카드
- 보유 카드 목록
- 태그 필터
- 정렬 제어
- Add Slot

규칙:
- 일반 보유 카드는 2행 3열 지표 배치를 유지한다.
- 모바일에서는 항목명 좌측, 값 우측 배치를 사용한다.
- 포트폴리오 요약 카드는 일반 보유 카드와 시각적으로 구분한다.

## Settings
목적:
- 연결된 시트 상태와 주요지표 비교 대상을 관리한다.

구성:
- 연결된 스프레드시트 정보
- 시트 행 수 정보
- Benchmark Targets 관리 영역

규칙:
- 기본 지표는 활성화/비활성화만 허용한다.
- custom 지표는 추가, 수정, 삭제를 허용한다.
- 미국 시장 티커만 허용한다.
- 동일 지수 또는 동일 티커 중복은 허용하지 않는다.
- custom 지표는 최대 3개까지 허용한다.

## Login
목적:
- Google 로그인 및 연결 상태 확인

구성:
- 단순 중앙 레이아웃
- 로그인 버튼
- Dev/Test 상태 문구

규칙:
- 사용자 불필요 문구는 줄이되, 개발/테스트에 필요한 문구는 `[Dev/Test]` 첨언과 함께 유지한다.
