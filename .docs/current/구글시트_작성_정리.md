# 구글시트 작성 정리

## 목적
- 현재 구현 기준의 Google Sheets 구조와 운영 규칙을 정리한다.
- 활성 구조와 후속 결정 항목을 분리한다.

## 현재 구현 구조
- `Holdings`
- `Monitor`
- `Benchmarks`
- `SeriesCalendar`
- `Series`

## 현재 프로젝트 전제
- 현재 프로젝트는 거래이력 저장 모델이 아니다.
- `Holdings`는 현재 보유 상태를 저장한다.
- holdings 입력은 `Add / Reduce` 기준의 현재 포지션 조정이다.
- 포트폴리오 시계열은 현재 보유 상태 기준 계산값이다.

## 탭별 역할
### Holdings
- 현재 보유 상태 저장
- 주요 컬럼: `ticker`, `name`, `side`, `quantity`, `avg_price`, `tags`, `display_order`

### Monitor
- holdings 계산과 benchmark 카드 계산용 가격 snapshot
- 주요 컬럼: `ticker`, `full_ticker`, `closeyest`, `ytd_price`, `price_1y`, `price_3y`, `price_5y`, `tradetime`

### Benchmarks
- 비교 대상 benchmark 정의와 조회 상태 저장
- 주요 컬럼: `benchmark_key`, `ticker_primary`, `ticker_fallback`, `resolved_ticker`, `resolved_source`, `status`, `market`, `name`, `category`, `accent_color`, `is_default`, `is_enabled`, `display_order`, `retry_count`

### SeriesCalendar
- 차트 날짜축 저장
- 주요 컬럼: `calendar_key`, `calendar_type`, `point_date`, `week_anchor`, `period_scope`, `updated_at`
- `YTD`는 `DAILY`
- `3Y`, `5Y`는 `LONG + WEEKLY`

### Series
- 차트용 portfolio / benchmark 시계열 저장
- 주요 컬럼: `series_key`, `series_type`, `ticker`, `name`, `sample_type`, `point_date`, `point_value`, `updated_at`
- 정규화 값은 저장하지 않고 렌더링 계층에서 계산한다.

## benchmark 운영 규칙
- 기본 benchmark는 `S&P 500`, `Nasdaq 100`, `Dow Jones`, `KOSPI`, `KOSDAQ`
- custom benchmark는 최대 3개
- custom benchmark는 현재 `추가 / 삭제`만 지원
- fallback까지 실패하면 `status=failed`
- `failed`여도 `is_enabled`는 자동으로 끄지 않음
- `failed` benchmark는 비교 계산과 차트 렌더링에서 제외

## 동기화 규칙
- 첫 연결과 자동 복구에서는 `Monitor`를 먼저 동기화
- `Series`는 background로 갱신
- 그래서 benchmark 카드는 차트보다 먼저 보일 수 있다
- holdings add/update/delete는 optimistic snapshot 후 background refresh를 사용한다

## 후속 결정 항목
- `KOSPI`, `KOSDAQ` fallback ticker
- fallback 문구 기준
- 오류 caption 문구 기준
- 포트폴리오 계산 검증 기준
- debug export 플래그 기준
