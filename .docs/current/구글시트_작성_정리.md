# 구글시트 작성 정리

## 목적
- Google Sheets 템플릿 작성과 확장 시트 설계를 한 문서에서 빠르게 확인할 수 있도록 정리한다.
- 현재 확정된 시트 구조와, 아직 결정이 필요한 항목을 분리해서 관리한다.

## 문서 사용 기준
- 확정된 구조와 규칙은 이 문서를 기준으로 작성한다.
- 세부 탭 설명은 `시트_템플릿_설계서.md`를 기준으로 유지한다.
- 아직 사용자 판단이 필요한 항목은 이 문서와 `decisions/지수비교_미결정사항.md`를 같이 확인한다.

## 현재 작성 기준
- 현재 운영 탭은 `Holdings`, `Watchlists`, `Monitor`, `Benchmarks`다.
- 현재 프로젝트는 거래이력 탭 없이 `현재 보유 상태` 중심으로 데이터를 관리한다.
- 그래프 단계 전까지 `Monitor`는 유지한다.
- 그래프 단계에서는 `Quotes`, `SeriesCalendar`, `Series`를 추가한다.
- 정규화 값은 시트에 저장하지 않고 데이터처리 계층에서 계산한다.
- fallback 시계열은 상시 적재하지 않는다.
- fallback까지 실패하면 해당 benchmark는 `failed` 상태로 저장한다.
- `failed` 상태가 되어도 `is_enabled`는 자동으로 바꾸지 않는다.
- `failed` 상태의 benchmark는 비교 계산과 차트 렌더링에서 자동 제외한다.

## 현재 운영 탭
### Holdings
목적:
- 보유 종목 원본 데이터 저장
- 현재 보유 상태를 저장한다.
- 개별 거래 체결 이력은 저장하지 않는다.

주요 컬럼:
- `ticker`
- `name`
- `quantity`
- `avg_price`
- `tags`
- `note`
- `display_order`
- `updated_at`

### Watchlists
목적:
- 관심 종목 원본 데이터 저장

주요 컬럼:
- `ticker`
- `name`
- `target_price`
- `tags`
- `note`
- `updated_at`

### Monitor
목적:
- 현재 Dashboard 비교 카드와 Holdings 계산용 가격 스냅샷 저장

주요 컬럼:
- `ticker`
- `market`
- `name`
- `current_price`
- `close_yesterday`
- `ytd_price`
- `price_1y`
- `price_3y`
- `price_5y`
- `updated_at`

규칙:
- 현재 Dashboard 비교 카드는 `Monitor` 스냅샷 값을 사용한다.
- 현재 Holdings 계산은 `Monitor`를 기준으로 한다.
- 그래프 단계 전까지 삭제하지 않는다.

### Benchmarks
목적:
- 비교 대상 벤치마크 정의와 조회 상태 관리

주요 컬럼:
- `benchmark_key`
- `ticker_primary`
- `ticker_fallback`
- `resolved_ticker`
- `resolved_source`
- `status`
- `market`
- `name`
- `category`
- `is_default`
- `is_enabled`
- `display_order`
- `retry_count`

확정 규칙:
- 기본 고정 세트는 `S&P 500`, `Nasdaq 100`, `Dow Jones`, `KOSPI`, `KOSDAQ`다.
- 사용자 추가 티커는 미국 시장만 허용한다.
- 사용자 추가 티커는 최대 3개다.
- 동일 지수 또는 동일 티커의 중복 설정은 허용하지 않는다.
- `ticker_primary`는 직접지수 기준 티커다.
- `ticker_fallback`은 fallback 후보 메타데이터다.
- 적절한 fallback 후보가 없으면 `ticker_fallback`은 비워둘 수 있다.
- 실제 조회는 `resolved_ticker` 하나만 사용한다.
- `resolved_source`는 `primary`, `fallback` 중 하나다.
- `status`는 `ready`, `retrying`, `fallback`, `failed` 중 하나다.
- primary 조회는 최대 2회 재시도한다.
- 재시도 후에도 실패하면 fallback이 있는 경우에만 전환한다.
- fallback까지 실패하면 `status`를 `failed`로 저장한다.
- `failed` 상태가 되어도 `is_enabled`는 자동으로 변경하지 않는다.
- `failed` 상태의 지표는 비교 계산과 차트 렌더링에서 자동 제외한다.

## 그래프 단계 확장 탭
### Quotes
목적:
- 그래프 단계 이후 카드 계산용 가격 스냅샷 구조

주요 컬럼:
- `ticker`
- `market`
- `name`
- `current_price`
- `close_yesterday`
- `ytd_price`
- `price_1y`
- `price_3y`
- `price_5y`
- `updated_at`

규칙:
- `Quotes`는 계산식 저장 또는 자동생성을 기준으로 한다.
- `Monitor`를 대체하거나 분리하는 다음 단계 구조다.

### SeriesCalendar
목적:
- 그래프와 포트폴리오 합성을 위한 기준 날짜축 관리

주요 컬럼:
- `calendar_key`
- `calendar_type`
- `point_date`
- `week_anchor`
- `period_scope`
- `updated_at`

규칙:
- `calendar_type`은 `DAILY`, `WEEKLY`다.
- `period_scope`는 `YTD`, `LONG`이다.
- `CalendarDaily`는 당해연도 거래일만 생성한다.
- `CalendarWeekly`는 최근 5년 범위의 주간 기준일만 생성한다.
- 주간 기준일은 금요일로 고정한다.
- `SeriesCalendar`는 앱 또는 동기화 루틴이 생성한다.

### Series
목적:
- 포트폴리오 및 벤치마크 비교용 시계열 저장

주요 컬럼:
- `series_key`
- `series_type`
- `ticker`
- `name`
- `sample_type`
- `point_date`
- `point_value`
- `updated_at`

규칙:
- `series_type`은 `portfolio`, `benchmark`다.
- `sample_type`은 `DAILY`, `WEEKLY`다.
- `DAILY`는 당해연도 범위만 저장하고 `YTD`에만 사용한다.
- `WEEKLY`는 최근 5년 범위만 저장하고 `3Y`, `5Y`에 사용한다.
- `Series`는 `SeriesCalendar` 날짜축에 맞춰 정렬한다.
- 기준축 날짜에 값이 없으면 가장 가까운 이전 유효값을 사용한다.
- 허용 간격은 `DAILY` 7일, `WEEKLY` 14일이다.
- 허용 간격을 초과하면 결측으로 처리한다.
- benchmark 시계열은 `resolved_ticker` 기준으로 적재한다.
- fallback 시계열은 상시 적재하지 않는다.
- fallback까지 실패하면 해당 benchmark는 `failed` 상태로 남긴다.
- `failed` 상태의 benchmark 시계열은 비교 계산에서 제외한다.
- 포트폴리오 시계열은 데이터처리 계층에서 계산한다.
- 현재 포트폴리오 시계열은 거래이력 기반이 아니라 현재 `Holdings` 상태 기준의 임시 계산이다.
- 따라서 과거 임의 시점 포트폴리오 값은 실제 거래 이력을 반영한 확정 성과가 아닐 수 있다.
- 정규화 값은 저장하지 않는다.

## 구글시트 작성 시 남은 결정 항목
### 우선 결정 필요
1. `Benchmarks`의 `KOSPI`, `KOSDAQ` fallback 후보 티커 확정

### 구현 직전 확정 필요
- fallback 사용 표시 문구 기준
- 오류 캡션 문구 기준
- 디버그 export 플래그 이름 및 위치
- 포트폴리오 계산 결과 검증 기준

## 작성 순서 권장안
1. 현재 운영 탭 `Holdings`, `Watchlists`, `Monitor`, `Benchmarks`부터 템플릿에 반영한다.
2. `Benchmarks`의 fallback 후보가 필요한 항목만 미결정으로 남긴다.
3. 그래프 착수 시점에 `Quotes`, `SeriesCalendar`, `Series`를 추가한다.
4. 차트 구현 전에 미결정 항목을 먼저 닫는다.
