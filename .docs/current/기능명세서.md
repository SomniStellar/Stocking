# 기능명세서

## 목적
Google Sheets 기반 개인 투자 관리 앱의 현재 기능 범위와 다음 구현 범위를 구분해서 관리한다.

## 현재 구현 범위
- Google 로그인 및 스프레드시트 연결
- Holdings / Watchlists 조회 및 편집
- Holdings 사용자 수동 정렬 및 드래그 앤 드롭 순서 변경
- Dashboard 요약 정보 표시
- Dashboard 주요지표 비교 카드 표시
- Settings 주요지표 비교 대상 관리

## 다음 구현 범위
- Dashboard 기간별 시계열 선형 그래프
- 비교용 시계열 저장 구조 확장
- 표시 통화 전환 기능

## 주요지표 비교 목표
- 동일 시작점을 기준으로, 현재 구성된 포트폴리오와 주요 지표가 현재 시점까지 어떻게 변화했는지 비교한다.
- 기본 비교 세트는 고정 세트를 사용한다.
- 사용자는 미국 시장 티커를 최대 3개까지 추가 비교할 수 있다.

## 비교 대상
### 기본 고정 세트
- S&P 500
- Nasdaq 100
- Dow Jones Industrial Average
- KOSPI
- KOSDAQ

### 사용자 추가 세트
- 미국 시장 티커만 허용한다.
- 추가 가능 개수는 최대 3개로 제한한다.
- 동일 지수 또는 동일 티커의 중복 설정은 허용하지 않는다.

## Dashboard 범위
- 현재는 비교 카드와 기간 토글만 구현한다.
- 기간 토글은 `YTD`, `1Y`, `3Y`, `5Y`를 사용한다.
- 기간별 시계열 선형 그래프는 다음 단계 범위로 둔다.
- 향후 다중 선택 비교 UI 확장 가능
- 향후 확대/축소 UI 확장 가능

## 현재 데이터 범위
- Holdings: 보유 종목 원본 데이터
- Watchlists: 관심 종목 원본 데이터
- Monitor: 카드 계산용 가격 스냅샷
- Benchmarks: 비교 대상 정의

## 다음 데이터 범위
- Quotes: 카드 계산용 가격 스냅샷 대체 구조
- SeriesCalendar: 그래프용 기준 날짜축
- Series: 포트폴리오 및 지표 비교용 시계열 데이터

## 현재 저장 및 계산 기준
- 현재 구현은 `Monitor`, `Benchmarks`를 사용한다.
- Dashboard 비교 카드는 `Monitor` 스냅샷 값으로 계산한다.
- Settings에서 기본 지표 활성화, custom 지표 추가, 수정, 삭제를 처리한다.

## 다음 저장 및 계산 기준
- `DAILY`는 당해연도 범위만 사용하며 `YTD` 전용이다.
- `WEEKLY`는 최근 5년 범위를 사용하며 `1Y`, `3Y`, `5Y` 전용이다.
- 화면은 저장된 원본 시계열에서 선택 기간 범위만 절단해서 사용한다.
- `Quotes`, `Benchmarks`, Benchmark 기준 시계열은 계산식 저장 / 자동생성을 기준으로 한다.
- `SeriesCalendar`는 앱/동기화 루틴이 생성한다.
- Portfolio 기준 시계열은 데이터처리 계층에서 계산한다.

## 벤치마크 조회 및 fallback 정책
- 직접지수를 primary 기준으로 사용한다.
- fallback은 해당 지수의 인덱스 펀드 또는 ETF를 후보로 둔다.
- 적절한 fallback 티커를 찾지 못한 경우 `ticker_fallback`은 비워둘 수 있다.
- 시트에는 fallback 후보 메타데이터만 저장하고, fallback 시계열을 상시 적재하지 않는다.
- 실제 조회는 `resolved_ticker` 하나만 사용한다.
- primary 조회 실패 시 최대 2회 재시도한다.
- 재시도 후에도 실패하면 `ticker_fallback`이 있는 경우에만 fallback으로 전환한다.
- fallback 사용 시 UI에 대체 지표 사용 사실을 표시한다.
- fallback도 실패하면 메시지를 표시하고 해당 지표는 카드/그래프 계산에서 제외한다.

## 포트폴리오 비교 기준
- 포트폴리오 시계열은 synthetic basket 방식으로 계산한다.
- 시작 시점은 비교 대상과 동일하게 맞춘다.
- 현재 구성된 포트폴리오를 동일 시작점에 놓고 현재 시점까지의 변화를 비교한다.
- 포트폴리오 시계열과 지표 시계열은 동일한 `Series` 구조를 사용한다.

## 정규화 정책
- 정규화 계산은 UI가 아니라 데이터처리 계층에서 수행한다.
- 정규화는 선택된 기간의 시작점 값을 기준으로 한다.
- 계산식은 `(point_value / base_value) * 100`이다.
- 시작점 이전에 유효값이 없으면 해당 시리즈는 해당 기간 비교에서 제외한다.

## 날짜축 정책
- `SeriesCalendar`를 기준 날짜축으로 사용한다.
- `CalendarDaily`는 당해연도 거래일 축을 사용한다.
- `CalendarWeekly`는 최근 5년 주간 축을 사용한다.
- `CalendarWeekly`의 기준일은 금요일이다.
- 기준축 날짜에 값이 없으면 가장 가까운 이전 유효값을 사용한다.
- 허용 간격은 `DAILY` 7일, `WEEKLY` 14일이다.
- 허용 간격을 초과하면 해당 시점은 결측으로 처리한다.

## 디버그 정책
- 실제 서비스 계산 경로는 하나만 유지한다.
- UI는 항상 데이터처리 계층의 계산 결과만 사용한다.
- 포트폴리오 시계열 디버깅이 필요하면 선택적 export를 둘 수 있다.
- 디버그 export는 기본값 `OFF`이며 실서비스 데이터 원천으로 사용하지 않는다.
- 디버그 export는 제거가 쉽도록 별도 플래그로 제어한다.
