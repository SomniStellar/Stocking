# Feature Spec
## Purpose
This document defines the active feature scope for the current Stocking build and separates implemented behavior from next-step work.

## Implemented Scope
- Google sign-in and spreadsheet connection flow
- Holdings read, edit, delete, reorder, and tag filtering
- Dashboard portfolio summary cards
- Dashboard benchmark comparison cards
- Dashboard benchmark comparison range chart
- Dashboard quick add benchmark action in the benchmark section header
- Dashboard benchmark enable or disable by clicking the card itself
- Dashboard benchmark toggle reflects immediately in the card UI and only requests monitor or series sync when ticker data is missing or changed
- Dashboard custom benchmark inline delete
- Dashboard benchmark drag-and-drop reorder
- Dashboard portfolio and benchmark accent color selection from a fixed palette
- Initial spreadsheet connect and reconnect load `Monitor` first and refresh `Series` in the background
- Settings for account and spreadsheet actions only

## Current Entry Points
- Active routed pages are `Login`, `Dashboard`, `Holdings`, and `Settings`.

## On Hold
- `DashboardComparisonChart` lazy mount or deferred mount is optional optimization work and is not required at the current stage.

## Next Scope
- Optional top-sector portfolio profit line chart
- Currency display toggle

## On Hold But Good To Have
- Add lazy mount or deferred mount to `DashboardComparisonChart` only if first paint becomes a clear issue again.
- Preserve chart shell height and avoid layout shift if lazy or deferred mount is introduced later.
- Finalize `KOSPI`, `KOSDAQ` fallback tickers.
- Finalize fallback copy and error caption copy.
- Finalize portfolio calculation verification rules.

## Dashboard Rules
- Portfolio summary stays on the main page.
- Benchmark comparison stays on the main page under the portfolio summary.
- Top summary uses 4 cards: Portfolio Value, Invested Cost, Unrealized P/L, and Return.
- Portfolio Value and Invested Cost cards do not keep helper caption text.
- Top summary cards use fixed width.
- Top summary wraps only between `1x4` and `2x2` based on available width.
- Top summary uses shared fixed-grid tokens from the common card layer.
- Benchmark comparison uses fixed-width cards and wraps by width as `4 -> 3 -> 2 -> 1`.
- Benchmark comparison uses shared fixed-grid tokens from the common card layer.
- Column reduction must happen before horizontal scrolling appears.
- Benchmark comparison uses a `Portfolio` reference card first.
- Comparison cards use a 3x2 grid.
- The `Portfolio` card uses the same card frame and row structure as benchmark cards, but keeps empty cells empty where no comparison label is needed.
- Benchmark cards use `benchmark name | empty` on row 1, `empty | benchmark return` on row 2, and `delta label | portfolio minus benchmark delta` on row 3.
- The current delta label copy is `vs Port.`.
- The delta is defined as `benchmark return - portfolio return`.
- Clicking a benchmark card toggles active or inactive state.
- Inactive cards remain visible but are desaturated and dimmed.
- Benchmark cards can be reordered by drag-and-drop, while the `Portfolio` card stays fixed in the first slot.
- Portfolio and benchmark accent colors can be selected only from a fixed palette.
- `Add Benchmark` lives in the benchmark section header and opens ticker-only quick input.
- The open quick add state stays inline in the title row as compact `Ticker + Save + Cancel`.
- Custom benchmark delete stays inline inside the same section.
- Period toggle stays `YTD`, `3Y`, `5Y`.
- Benchmark comparison chart is directly below the benchmark cards.
- Benchmark comparison chart width follows the same centered fixed-width rule as the benchmark comparison card grid.
- Period toggle changes the whole benchmark comparison chart range.
- The benchmark comparison chart is range-based, not a simultaneous period summary chart.
- A separate top-sector portfolio profit chart is a follow-up option, not part of the current required scope.
- On first spreadsheet load, benchmark cards should appear before series-based chart data is fully synchronized.

## Holdings Rules
- Holdings layout is split into a centered top sector and a lower card sector.
- The top sector contains only the portfolio summary card and the add holding card.
- The low sector contains only holding cards.
- Both sectors use the same width.
- All cards use the same fixed size.
- The number of cards shown on one row depends on the fixed card width.
- Holdings uses shared fixed-grid tokens from the common card layer.
- Default holding card layout is 2 rows by 3 columns.
- `Portfolio` card structure is fixed.
  - Row 1 column 1: `Portfolio`
  - Row 1 column 2: `Value`
  - Row 1 column 3: 3-row `P/L` block with label, profit amount, return
  - Row 2 column 1: `Holdings`
  - Row 2 column 2: `Invested`
  - Row 2 column 3: internal 3-row by 2-column period block for `YTD`, `3Y`, `5Y`
- `Add Holding` card structure is fixed.
  - Row 1 column 1: `Ticker`
  - Row 1 column 2: one-line `Total Price` <-> `Avg Price` clickable toggle and price input
  - Row 1 column 3: `Add` <-> `Reduce` segmented control for current position adjustment
  - Row 2 column 1: `Quantity`
  - Row 2 column 2: `Tags`
  - Row 2 column 3: `Save`
- Holdings input adjusts the current position state directly and is not a transaction-history record editor.
- Price values use up to 2 decimal places.
- Quantity values use up to 4 decimal places and should not show trailing zero noise.
- Switching `Total Price` and `Avg Price` in holding edit keeps the stored holding basis stable and should not change the effective total by display rounding alone.
- Holding card structure is fixed.
  - Row 1 column 1: `Name`, `Ticker`
  - Row 1 column 2: `Price / Avg`, `Price`, `Avg`
  - Row 1 column 3: `P/L`, profit amount, return
  - Row 2 column 1: `Quantity`
  - Row 2 column 2: `Value / Inv.`, `Value`, `Invested`
  - Row 2 column 3: internal 3-row by 2-column period block for `YTD`, `3Y`, `5Y`
- Tags are case-insensitive.
- Average Price and Total Price input mode toggle must remain visibly clickable.

## Benchmark Target Rules
### Default set
- S&P 500
- Nasdaq 100
- Dow Jones Industrial Average
- KOSPI
- KOSDAQ

### Custom set
- Only US custom benchmark tickers are allowed.
- Up to 3 custom benchmarks are allowed.
- Duplicate benchmark keys are not allowed.
- Duplicate tickers are not allowed.

## Fallback Rules
- Primary ticker is the direct index target.
- Fallback ticker is optional metadata only.
- The sheet keeps fallback candidate metadata, not full fallback series data.
- Retry primary up to 2 times.
- If primary fails and fallback exists, resolve to fallback.
- If fallback also fails, exclude the series and show a status caption.

## Current Data Structure
- `Holdings`
- `Monitor`
- `Benchmarks`
- `SeriesCalendar`
- `Series`

## Planned Data Structure
- `Quotes`

## Settings Rules
- Settings keeps only account status, sign-in or sign-out, sheet creation or sheet connection, sheet sync, sheet rebuild, open sheet, and disconnect sheet actions.
- Settings does not expose internal sheet row counts or snapshot counters in the default UI.
- The global topbar does not show a persistent spreadsheet connection status chip.
