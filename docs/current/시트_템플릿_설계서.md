# 시트 템플릿 설계서

## 목적
- 현재 구현 기준 Google Sheets 템플릿 구조를 정의한다.
- 활성 탭과 주요 컬럼을 빠르게 확인할 수 있게 한다.

## 현재 기준 탭
- `Holdings`
- `Monitor`
- `Benchmarks`
- `SeriesCalendar`
- `Series`

## 전제
- 현재 프로젝트는 거래이력 모델이 아니다.
- `Holdings`는 현재 보유 상태를 저장한다.
- holdings 입력은 현재 포지션 증감 입력이다.
- 포트폴리오 시계열은 현재 보유 상태 기준 계산값이다.

## Holdings
목적:
- 현재 보유 종목 저장
주요 컬럼:
- `ticker`
- `name`
- `side`
- `quantity`
- `avg_price`
- `tags`
- `display_order`

## Monitor
목적:
- holdings 계산과 dashboard benchmark 카드 계산용 가격 snapshot 저장
주요 컬럼:
- `ticker`
- `full_ticker`
- `closeyest`
- `ytd_price`
- `price_1y`
- `price_3y`
- `price_5y`
- `tradetime`

## Benchmarks
목적:
- benchmark 정의와 상태 저장
주요 컬럼:
- `benchmark_key`
- `ticker_primary`
- `ticker_fallback`
- `resolved_ticker`
- `resolved_source`
- `status`
- `market`
- `name`
- `category`
- `accent_color`
- `is_default`
- `is_enabled`
- `display_order`
- `retry_count`

규칙:
- 기본 benchmark 세트는 `S&P 500`, `Nasdaq 100`, `Dow Jones`, `KOSPI`, `KOSDAQ`
- custom benchmark는 최대 3개
- fallback까지 실패하면 `status=failed`
- `failed`여도 `is_enabled`는 자동 변경하지 않음

## SeriesCalendar
목적:
- chart range용 날짜축 저장
주요 컬럼:
- `calendar_key`
- `calendar_type`
- `point_date`
- `week_anchor`
- `period_scope`
- `updated_at`

규칙:
- `YTD`는 `DAILY`
- 장기 범위는 `WEEKLY`
- 미완료 당일 데이터는 제외

## Series
목적:
- portfolio / benchmark chart용 시계열 저장
주요 컬럼:
- `series_key`
- `series_type`
- `ticker`
- `name`
- `sample_type`
- `point_date`
- `point_value`
- `updated_at`

규칙:
- `series_type`은 `portfolio` 또는 `benchmark`
- 정규화 값은 저장하지 않음
- `failed` benchmark는 chart 계산에서 제외

## 현재 후속 항목
- `Quotes` 도입 여부
- `KOSPI`, `KOSDAQ` fallback ticker 확정
- 포트폴리오 계산 검증 기준 정리
