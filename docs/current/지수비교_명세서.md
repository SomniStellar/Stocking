# 지수비교 명세서

## 목적
- 현재 Stocking 빌드에서 사용하는 benchmark comparison 규칙을 한 문서에서 확인한다.
- 완료된 규칙만 다루고, 미결정 항목은 별도 문서에 남긴다.

## 현재 범위
- Dashboard에서 benchmark comparison 카드와 range chart를 제공한다.
- 비교 기간은 `YTD`, `3Y`, `5Y`를 사용한다.
- 차트는 기간 버튼에 따라 전체 범위를 전환한다.
- 차트 1차 구현은 `Recharts`를 사용한다.

## 기본 benchmark
- `S&P 500`
- `Nasdaq 100`
- `Dow Jones`
- `KOSPI`
- `KOSDAQ`

## custom benchmark
- 미국 시장 ticker만 허용한다.
- 최대 3개까지 허용한다.
- 중복 benchmark key는 허용하지 않는다.
- 중복 ticker는 허용하지 않는다.
- 현재 custom benchmark 카드는 `추가/삭제`만 지원한다.

## 카드 규칙
- 첫 비교 카드는 항상 `Portfolio`다.
- benchmark 카드는 `4 -> 3 -> 2 -> 1` 열로 줄어든다.
- 카드 클릭으로 활성/비활성을 전환한다.
- 비활성 상태 카드는 화면에는 남기고, 색을 약화한다.
- 카드 순서는 drag-and-drop으로 바꿀 수 있다.
- `Portfolio` 카드는 고정이며 reorder 대상이 아니다.
- accent color는 고정 팔레트 안에서 선택한다.

## delta 규칙
- 카드의 `vs Port.` 값은 `benchmark return - portfolio return`이다.
- delta 값은 `%p` 기준으로 표시한다.
- 부호에 따라 색을 달리 표시한다.

## 차트 규칙
- 차트는 benchmark 카드 아래에 배치한다.
- 차트 폭은 benchmark 카드 grid와 같은 centered fixed-width 규칙을 따른다.
- 차트는 `YTD`, `3Y`, `5Y`를 동시에 요약하지 않고, 선택된 기간 하나만 렌더링한다.
- hover는 현재 선택 범위 내 해당 시점 값을 보여준다.
- 초기 live 로딩에서는 카드가 먼저 보이고, series sync가 끝난 뒤 차트가 보강될 수 있다.

## 데이터 구조
- 현재 live 구조는 `Holdings`, `Monitor`, `Benchmarks`, `SeriesCalendar`, `Series`를 사용한다.
- portfolio는 현재 보유 상태 기반 모델이다.
- portfolio series는 현재 `Holdings` 기준 임시 계산값을 사용한다.
- 정규화는 저장이 아니라 렌더링 계층에서 계산한다.

## 조회 규칙
- primary ticker를 먼저 조회한다.
- fallback ticker는 선택적 메타데이터다.
- 시트에는 fallback 후보 정보만 저장한다.
- primary는 최대 2회까지 재시도한다.
- primary 실패 후 fallback이 있으면 fallback으로 전환한다.
- fallback까지 실패하면 `status=failed`로 둔다.
- `failed` 상태는 `is_enabled`를 자동으로 끄지 않는다.

## 실패 처리
- fallback 사용 시 대체 지표 사용 상태를 표시한다.
- fallback까지 실패하면 오류 캡션을 표시한다.
- `failed` 상태 benchmark는 카드에는 남고, 비교 계산과 차트에서는 제외한다.

## 날짜축 규칙
- `YTD`는 `DAILY` 축을 사용한다.
- `3Y`, `5Y`는 `WEEKLY` 축을 사용한다.
- 일봉은 마지막 완료 영업일까지 사용한다.
- 주봉은 마지막 완료 금요일까지 사용한다.
- 값이 없는 날짜는 허용 범위 안에서 직전 유효값을 사용한다.

## 후속 항목
- benchmark toggle 반응속도 개선 완료
- `DashboardComparisonChart` lazy 또는 deferred mount는 현재 단계에서는 보류
- fallback 문구와 fallback ticker 확정은 미결정 문서에서 계속 관리
