# 화면별 상세 UI 명세서 v1

## 1. 문서 목적

본 문서는 `기능명세서_v1.md`를 기준으로 실제 화면 구현에 필요한 UI 구조, 컴포넌트, 상태, 사용자 액션을 정리한 개발 참조 문서이다.

본 문서는 다음에 활용한다.

- 페이지 구조 설계
- 컴포넌트 분해 기준 설정
- 상태 관리 범위 정의
- 개발 우선순위 정리

## 2. 전체 화면 구성

MVP 화면은 아래 6개로 구성한다.

- 로그인 화면
- 대시보드 화면
- 보유 포트폴리오 화면
- 즐겨찾기 화면
- 가상 포트폴리오 화면
- 설정 화면

공통 레이아웃 구조:

- 상단 헤더
- 좌측 또는 상단 내비게이션
- 본문 컨텐츠 영역
- 로딩/오류 상태 영역

## 3. 공통 UI 원칙

- 데스크톱 우선으로 구성하되 모바일에서도 사용 가능해야 한다.
- 표 중심 화면은 모바일에서 카드형 요약으로 일부 전환 가능해야 한다.
- 금액/수익률은 가독성 높은 형태로 표시한다.
- 상승/하락 값은 색상으로 구분한다.
- 데이터 없음 상태를 명확히 보여준다.
- Google Sheets 연결 실패나 종목 조회 실패는 사용자에게 이해 가능한 문구로 표시한다.

## 4. 로그인 화면

### 목적

사용자의 진입과 Google 계정 인증을 처리한다.

### 주요 UI 요소

- 서비스 타이틀
- 서비스 설명 문구
- `Google로 로그인` 버튼
- 연동 방식 안내 문구
- 오류 메시지 박스

### 상태

- 기본 상태
- 로그인 진행 중
- 로그인 실패

### 사용자 액션

- Google 로그인 버튼 클릭

### 처리 결과

- 성공 시 대시보드 또는 초기 연결 화면으로 이동
- 실패 시 오류 메시지 표시

## 5. 대시보드 화면

### 목적

전체 상태를 한 화면에서 요약한다.

### 상단 요약 카드

표시 카드:

- 총 평가금액
- 총 매입금액
- 총 손익
- 총 수익률
- 보유 종목 수
- 즐겨찾기 종목 수
- 가상 포트폴리오 수

### 주요 섹션

#### 보유 포트폴리오 요약

- 상위 손익 종목 5개
- 수익률 상위 종목 5개
- 손실 종목 요약

#### 즐겨찾기 요약

- 목표가 근접 종목
- 전일 상승 종목
- 전일 하락 종목

#### 가상 포트폴리오 요약

- 포트폴리오별 수익률 카드
- 포트폴리오별 평가금액 카드

### 상태

- 데이터 로딩 중
- 데이터 없음
- 정상 조회

## 6. 보유 포트폴리오 화면

### 목적

실제 보유 종목을 조회하고 수정한다.

### 주요 UI 요소

- 페이지 제목
- `종목 추가` 버튼
- 정렬 드롭다운
- 필터 입력창
- 보유 목록 테이블
- 종목 추가/수정 모달
- 삭제 확인 모달

### 테이블 컬럼

- 종목코드
- 종목명
- 보유수량
- 평균단가
- 전일 종가
- 평가금액
- 손익
- 수익률
- 전일 대비 금액
- 전일 대비 비율
- 52주 위치율
- 시가총액
- PER
- EPS
- 액션

### 액션 버튼

- 수정
- 삭제

### 입력 폼 항목

- `ticker`
- `quantity`
- `avg_price`
- `memo`

### 유효성 검증

- `ticker` 필수
- `quantity > 0`
- `avg_price > 0`

### 빈 상태 문구 예시

- 등록된 보유 종목이 없습니다.
- 종목을 추가해 포트폴리오를 시작하세요.

## 7. 즐겨찾기 화면

### 목적

관심 종목을 모니터링한다.

### 주요 UI 요소

- 페이지 제목
- `즐겨찾기 추가` 버튼
- 목표가 필터
- 즐겨찾기 목록 테이블
- 추가/수정 모달
- 삭제 확인 모달

### 테이블 컬럼

- 종목코드
- 종목명
- 전일 종가
- 전일 대비 비율
- 목표가
- 목표가 대비 차이
- 목표가 대비 괴리율
- 52주 위치율
- 메모
- 액션

### 입력 폼 항목

- `ticker`
- `target_price`
- `memo`

### 보조 표시

- 목표가 근접 배지
- 목표가 상회/하회 상태 텍스트

### 빈 상태 문구 예시

- 즐겨찾기 종목이 없습니다.
- 관심 있는 미국 주식을 추가해보세요.

보유 외 종목 조합을 저장하고 성과를 확인한다.

- 페이지 제목

- `종목 추가` 버튼
- 포트폴리오 선택 또는 그룹 표시
- 목록 테이블
- 생성/수정 모달
- 삭제 확인 모달

### 테이블 컬럼

- 포트폴리오명
- 종목코드
- 종목명
- 가상 수량
- 가상 매수가
- 전일 종가
- 가상 평가금액
- 가상 손익
- 가상 수익률
- 메모
- 액션

### 입력 폼 항목

- `portfolio_name`
- `ticker`
- `virtual_qty`
- `virtual_entry_price`
- `memo`

### 유효성 검증

- `portfolio_name` 필수
- `ticker` 필수

- `virtual_entry_price > 0`

### 요약 영역

- 포트폴리오별 총 손익
- 포트폴리오별 총 수익률

## 9. 설정 화면

### 목적

계정과 스프레드시트 연결 상태를 확인하고 재설정한다.

### 주요 UI 요소

- 연결된 Google 계정 카드
- 연결된 Spreadsheet 정보 카드
- `재연결` 버튼
- 시트 구조 안내 섹션
- `로그아웃` 버튼

### 표시 항목

- 사용자 이메일 또는 계정 식별자
- 현재 연결된 스프레드시트 이름 또는 ID
- 마지막 동기화 시각

## 10. 종목 등록 UX

### 입력 방식

- 사용자가 미국 주식 `ticker`를 직접 입력한다.
- 예: `AAPL`, `MSFT`, `NVDA`, `KO`

### 동작 흐름

1. 사용자가 `ticker` 입력
2. 앱이 대문자로 정규화
3. `GOOGLEFINANCE` 조회 가능 여부 확인
4. 성공 시 `Stocks`에 등록 가능
5. 실패 시 오류 메시지 표시

### 오류 메시지 예시

- 조회 가능한 미국 주식 종목이 아닙니다.
- 이미 등록된 종목입니다.

## 13. 반응형 처리 기준

데스크톱:

- 표 중심 레이아웃 사용
- 3~4열 요약 카드 구성

모바일:

- 요약 카드는 1열 또는 2열
- 표는 중요 컬럼만 남기고 일부 축약
- 상세 값은 카드/아코디언 형태로 대체 가능

## 14. 우선 개발 순서

1. 로그인 화면
2. 공통 레이아웃
3. 설정 화면의 시트 연결 상태 확인
4. 보유 포트폴리오 화면
5. 즐겨찾기 화면
6. 가상 포트폴리오 화면
7. 대시보드 집계 화면
8. 모바일 최적화

## 15. 개발 메모

- 화면은 계산 결과를 많이 보여주지만, 계산 자체는 가능한 한 앱 로직에서 처리한다.
- 시트 데이터가 비어 있을 가능성이 높으므로 빈 상태 UI를 먼저 준비한다.
- MVP에서는 세련된 검색 자동완성보다 안정적인 `ticker 입력 + 검증` 흐름이 더 중요하다.
