# 기능명세서 v1

## 1. 문서 개요

본 문서는 `GitHub Pages + Google Sheets + GOOGLEFINANCE` 기반의 미국 주식 포트폴리오 모니터링 웹앱 개발을 위한 기준 설계서이다.

본 문서는 실제 개발 시 다음 목적에 사용한다.

- 구현 범위 고정
- 화면/데이터 구조 기준 정의
- 계산 규칙 기준 정의
- 추후 상세 설계 및 개발 작업의 참조 문서

## 2. 프로젝트 목표

사용자가 직접 입력한 미국 주식 종목만 대상으로 다음 기능을 제공한다.

- 보유 포트폴리오 관리
- 즐겨찾기 종목 모니터링
- 보유 외 가상 포트폴리오 관리
- 전일 종가 기준 주가/지표 모니터링

본 서비스는 서버 없이 정적 웹으로 운영하며, 데이터 저장과 조회는 Google 생태계 안에서 처리한다.

## 3. 확정된 제약 조건

### 3.1 데이터 및 인프라

- 프론트엔드는 `GitHub Pages`에 배포한다.
- 데이터 저장소는 사용자 `Google Spreadsheet`를 사용한다.
- 외부 주가 API는 사용하지 않는다.
- 주가 정보는 `Google Sheets`의 `GOOGLEFINANCE` 함수만 사용한다.

### 3.2 시장 범위

- 지원 시장은 `미국 주식`으로 한정한다.
- MVP에서는 사용자가 직접 `ticker`를 입력해 종목을 등록한다.
- 시장 전체 자동 검색/자동완성은 MVP 범위에서 제외한다.

### 3.3 가격 기준

- 실시간 시세는 사용하지 않는다.
- 모든 가격 기준은 `전일 종가(closeyest)`를 사용한다.
- 장중 변동 추적은 MVP 범위에서 제외한다.

### 3.4 제외 기능

- 배당 정보
- 외부 API 연동
- 실시간 시세
- 자동 매매 기능
- 다중 국가 시장 지원
- 소셜 기능

## 4. 시스템 개요

### 4.1 아키텍처

- 프론트엔드: `React + TypeScript + Vite`
- 배포: `GitHub Pages`
- 인증: `Google 로그인`
- 데이터 저장: 사용자 `Google Spreadsheet`
- 주가 조회: `GOOGLEFINANCE`

### 4.2 동작 방식

1. 사용자가 웹앱에 접속한다.
2. Google 계정으로 로그인한다.
3. 사용자의 Google Spreadsheet를 연결한다.
4. 사용자가 미국 주식 종목을 직접 등록한다.
5. 앱은 연결된 시트 데이터를 읽고 `GOOGLEFINANCE` 기반 모니터링 정보를 표시한다.
6. 사용자는 보유 포트폴리오, 즐겨찾기, 가상 포트폴리오를 관리한다.

## 5. 사용자 시나리오

### 5.1 보유 포트폴리오 사용자 시나리오

1. 사용자가 종목 코드를 입력해 종목을 등록한다.
2. 보유 화면에서 수량과 평균단가를 입력한다.
3. 시스템은 전일 종가 기준으로 평가금액, 손익, 수익률을 계산한다.
4. 사용자는 대시보드와 포트폴리오 화면에서 결과를 확인한다.

### 5.2 즐겨찾기 사용자 시나리오

1. 사용자가 관심 종목을 즐겨찾기에 추가한다.
2. 목표가와 메모를 입력한다.
3. 시스템은 전일 종가 기준으로 괴리율을 계산해 표시한다.

### 5.3 가상 포트폴리오 사용자 시나리오

1. 사용자가 실제 보유하지 않은 종목으로 가상 포트폴리오를 만든다.
2. 종목별 가상 수량과 가상 매수가를 입력한다.
3. 시스템은 전일 종가 기준으로 가상 손익과 수익률을 계산한다.

## 9. 주가/지표 조회 규칙

### 9.1 가격 기준

- 현재가처럼 표시하는 값은 실제로 `closeyest`를 사용한다.
- 모든 손익/수익률 계산은 `closeyest` 기준으로 수행한다.

### 9.2 조회 항목

다음 항목을 기본 모니터링 지표로 사용한다.

- `closeyest`

- `changepct`
- `high52`
- `low52`
- `marketcap`

- `eps`
- `volumeavg`

## 10. 계산 규칙

### 10.1 보유 포트폴리오

평가금액:

```text
quantity * closeyest
```

손익:

```text
(closeyest - avg_price) * quantity
```

수익률:

```text
((closeyest - avg_price) / avg_price) * 100
```

### 10.2 가상 포트폴리오

가상 평가금액:

```text
virtual_qty * closeyest
```

가상 손익:

```text
(closeyest - virtual_entry_price) * virtual_qty
```

가상 수익률:

```text
((closeyest - virtual_entry_price) / virtual_entry_price) * 100
```

### 10.3 보조 지표

52주 위치율:

```text
(closeyest - low52) / (high52 - low52)
```

목표가 대비 차이:

```text
target_price - closeyest
```

목표가 대비 괴리율:

```text
((target_price - closeyest) / closeyest) * 100
```

## 11. 종목 등록 규칙

### 11.1 입력 규칙

- 사용자는 종목코드를 직접 입력한다.
- 입력 예: `AAPL`, `MSFT`, `NVDA`, `KO`
- 입력값은 저장 시 대문자로 정규화한다.

### 11.2 검증 규칙

- 시스템은 입력된 종목코드가 미국 주식 조회 가능 종목인지 검증해야 한다.
- 검증은 `GOOGLEFINANCE` 조회 결과 유무를 기반으로 처리한다.
- 조회 실패 시 등록 불가 상태로 처리한다.

### 11.3 중복 규칙

- 이미 등록된 `ticker`는 중복 등록할 수 없다.

## 12. 대시보드 집계 규칙

### 12.1 보유 포트폴리오 집계

- 총 평가금액 = 보유 종목 평가금액 합계
- 총 매입금액 = `quantity * avg_price` 합계
- 총 손익 = 평가금액 합계 - 매입금액 합계
- 총 수익률 = 총 손익 / 총 매입금액

### 12.2 즐겨찾기 집계

- 즐겨찾기 종목 수 표시
- 목표가 근접 종목 표시
- 전일 대비 상승/하락 종목 요약 표시

### 12.3 가상 포트폴리오 집계

- 포트폴리오별 평가금액 합계
- 포트폴리오별 손익 합계
- 포트폴리오별 수익률 표시

## 13. 오류 처리 명세

### 13.1 인증 오류

- Google 로그인 실패 시 재시도 안내 메시지를 표시한다.
- 권한 거부 시 접근 권한 필요 메시지를 표시한다.

### 13.2 데이터 오류

- 시트가 존재하지 않거나 구조가 다른 경우 안내 메시지를 표시한다.
- 필수 컬럼이 누락된 경우 데이터 구조 오류로 표시한다.

### 13.3 주가 조회 오류

- `GOOGLEFINANCE` 조회 실패 시 `데이터 없음` 상태로 표시한다.
- 일부 지표가 비어 있어도 전체 화면이 중단되지 않아야 한다.

## 14. 비기능 요구사항

- `GitHub Pages`에서 정상 동작해야 한다.
- 데스크톱 및 모바일 브라우저에서 사용 가능해야 한다.
- 데이터 로딩 상태를 표시해야 한다.
- 시트의 일부 데이터가 비어 있어도 앱이 비정상 종료되지 않아야 한다.
- 소규모 종목 수 기준에서 빠르게 동작해야 한다.
