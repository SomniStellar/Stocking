# 시트 템플릿 설계서 v1

## 1. 문서 목적

본 문서는 `기능명세서_v1.md`를 기준으로 Google Spreadsheet의 실제 시트 탭 구조, 컬럼 정의, 입력 규칙, 수식 구성 원칙을 정리한 개발 참조 문서이다.

대상 서비스는 다음 조건을 따른다.

- `GitHub Pages + Google Sheets + GOOGLEFINANCE`
- `미국 주식 한정`
- `전일 종가(closeyest)` 기준 모니터링
- `보유 / 즐겨찾기 / 가상 포트폴리오` 지원
- `배당 기능 제외`

## 2. 스프레드시트 구성

사용자 Spreadsheet는 아래 5개 탭으로 구성한다.

- `Stocks`
- `Holdings`
- `Favorites`
- `Ideas`
- `Monitor`

권장 순서는 위와 같다.

## 3. 탭별 설계

## 3.1 `Stocks`

### 목적

사용자가 등록한 미국 주식의 마스터 목록을 관리한다.

### 컬럼 정의

| 컬럼명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| `ticker` | string | Y | 종목코드. 예: `AAPL` |
| `name` | string | N | 종목명 |
| `market` | string | Y | 기본값 `US` |
| `active` | boolean/string | Y | 사용 여부. `TRUE/FALSE` 또는 `Y/N` |
| `memo` | string | N | 사용자 메모 |

### 입력 규칙

- `ticker`는 대문자 저장
- 중복 `ticker` 금지
- `market`은 MVP에서 모두 `US`
- `active=TRUE`인 종목만 모니터링 대상

### 예시 데이터

| ticker | name | market | active | memo |
| --- | --- | --- | --- | --- |
| AAPL | Apple Inc. | US | TRUE | 아이폰 생태계 |
| MSFT | Microsoft Corp. | US | TRUE | 클라우드 |
| NVDA | NVIDIA Corp. | US | TRUE | AI 반도체 |

## 3.2 `Holdings`

### 목적

실제 보유 포트폴리오를 저장한다.

### 컬럼 정의

| 컬럼명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| `ticker` | string | Y | `Stocks.ticker` 참조 |
| `quantity` | number | Y | 보유 수량 |
| `avg_price` | number | Y | 평균단가 |
| `memo` | string | N | 메모 |

### 입력 규칙

- `ticker`는 `Stocks`에 등록된 값만 허용
- `quantity > 0`
- `avg_price > 0`

### 예시 데이터

| ticker | quantity | avg_price | memo |
| --- | --- | --- | --- |
| AAPL | 10 | 185.25 | 장기보유 |
| MSFT | 5 | 402.10 | 분할매수 |

## 3.3 `Favorites`

### 목적

보유 여부와 무관한 관심 종목을 저장한다.

### 컬럼 정의

| 컬럼명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| `ticker` | string | Y | `Stocks.ticker` 참조 |
| `target_price` | number | N | 목표가 |
| `memo` | string | N | 메모 |

### 입력 규칙

- `ticker`는 `Stocks`에 등록된 값만 허용
- 목표가는 비어 있어도 됨

### 예시 데이터

| ticker | target_price | memo |
| --- | --- | --- |
| NVDA | 950 | 눌림목 관찰 |
| AMZN | 210 | 클라우드 추적 |

## 3.4 `Ideas`

### 목적

보유 외 가상 포트폴리오를 저장한다.

### 컬럼 정의

| 컬럼명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| `portfolio_name` | string | Y | 가상 포트폴리오명 |
| `ticker` | string | Y | `Stocks.ticker` 참조 |
| `virtual_qty` | number | Y | 가상 수량 |
| `virtual_entry_price` | number | Y | 가상 매수가 |
| `memo` | string | N | 메모 |

### 입력 규칙

- `portfolio_name` 필수
- `ticker`는 `Stocks`에 등록된 값만 허용
- `virtual_qty > 0`
- `virtual_entry_price > 0`

### 예시 데이터

| portfolio_name | ticker | virtual_qty | virtual_entry_price | memo |
| --- | --- | --- | --- | --- |
| AI Growth | NVDA | 3 | 880 | 고성장 |
| Big Tech | AMZN | 4 | 182 | 장기 관찰 |

## 3.5 `Monitor`

### 목적

`GOOGLEFINANCE`를 통해 조회한 가격 및 지표를 집계한다.

### 컬럼 정의

| 컬럼명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| `ticker` | string | Y | 종목코드 |
| `full_ticker` | string | Y | 조회용 티커 |
| `closeyest` | number | N | 전일 종가 |
| `change` | number | N | 전일 대비 금액 |
| `changepct` | number | N | 전일 대비 비율 |
| `high52` | number | N | 52주 고가 |
| `low52` | number | N | 52주 저가 |
| `marketcap` | number | N | 시가총액 |
| `pe` | number | N | PER |
| `eps` | number | N | EPS |
| `volumeavg` | number | N | 평균 거래량 |
| `tradetime` | datetime/text | N | 거래 시각 |

### 생성 규칙

- `Stocks`의 `active=TRUE` 종목을 기준으로 생성
- `full_ticker`는 우선 `NASDAQ:` 접두어로 조회 시도
- 필요 시 향후 미국 시장 내 거래소 보완 규칙 추가 가능

### 기본 수식 예시

`ticker`:

```gs
=FILTER(Stocks!A2:A, Stocks!D2:D=TRUE)
```

`full_ticker`:

```gs
="NASDAQ:"&A2
```

`closeyest`:

```gs
=GOOGLEFINANCE(B2,"closeyest")
```

`change`:

```gs
=GOOGLEFINANCE(B2,"change")
```

`changepct`:

```gs
=GOOGLEFINANCE(B2,"changepct")
```

`high52`:

```gs
=GOOGLEFINANCE(B2,"high52")
```

`low52`:

```gs
=GOOGLEFINANCE(B2,"low52")
```

`marketcap`:

```gs
=GOOGLEFINANCE(B2,"marketcap")
```

`pe`:

```gs
=GOOGLEFINANCE(B2,"pe")
```

`eps`:

```gs
=GOOGLEFINANCE(B2,"eps")
```

`volumeavg`:

```gs
=GOOGLEFINANCE(B2,"volumeavg")
```

## 4. 앱 계산용 파생값

이 값들은 시트에 직접 저장하지 않고 앱에서 계산하는 것을 권장한다.

### 보유 포트폴리오

- 평가금액 = `quantity * closeyest`
- 손익 = `(closeyest - avg_price) * quantity`
- 수익률 = `((closeyest - avg_price) / avg_price) * 100`

### 가상 포트폴리오

- 가상 평가금액 = `virtual_qty * closeyest`
- 가상 손익 = `(closeyest - virtual_entry_price) * virtual_qty`
- 가상 수익률 = `((closeyest - virtual_entry_price) / virtual_entry_price) * 100`

### 관심종목

- 목표가 대비 차이 = `target_price - closeyest`
- 목표가 대비 괴리율 = `((target_price - closeyest) / closeyest) * 100`

### 공통 지표

- 52주 위치율 = `(closeyest - low52) / (high52 - low52)`

## 5. 유효성 검증 규칙

### 공통

- 숫자 컬럼은 숫자만 허용
- `ticker`는 대문자 영문 기준 저장
- 빈 행은 무시 가능해야 함

### 종목 유효성

- 앱에서 종목 등록 시 `GOOGLEFINANCE` 조회 성공 여부를 확인
- 조회 실패 시 등록 거부

### 관계 무결성

- `Holdings`, `Favorites`, `Ideas`의 `ticker`는 `Stocks`에 존재해야 함

## 6. 앱 연동 규칙

- 앱은 각 시트를 읽어 메모리에서 조합한다.
- 화면 표시를 위한 계산값은 가능하면 앱에서 계산한다.
- 시트는 원본 데이터 저장소 역할에 집중한다.
- `Monitor` 시트는 가격 및 핵심 지표 조회용 캐시성 구조로 사용한다.

## 7. 시트 초기 템플릿 작성 순서

1. `Stocks` 생성
2. `Holdings` 생성
3. `Favorites` 생성
4. `Ideas` 생성
5. `Monitor` 생성
6. 헤더행 고정
7. 컬럼명 입력
8. 필요 시 샘플 데이터 2~3건 입력

## 8. 개발 메모

- 시트 구조가 바뀌면 앱이 깨질 수 있으므로 컬럼명 기반 매핑을 고려한다.
- `Monitor`는 화면 렌더링 전에 값이 비어 있을 수 있으므로 빈 상태 처리가 필요하다.
- 미국 주식 한정이지만 `NASDAQ:` 고정 조회가 모든 종목에 항상 맞는 것은 아니므로, 추후 거래소 매핑 컬럼 확장을 고려한다.
